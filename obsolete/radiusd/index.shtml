Realm-based base DN selection (radiusd-cistron-1.6.4-ldaprealms-1.0.patch)

This patch adds realm-based base DN selection to Cistron radiusd. Basically, this patch carves your LDAP tree up into separate areas so you can host several different realms, each realm having its own discrete namespace. For example, you could have a realm foo.com with user joe (joe@foo.com) and a realm bar.com also with user joe (joe@bar.com). The two 'joe' users are two entirely separate accounts.

For example, if you host foo.com and bar.com, you would store user entries for foo.com under ou=foo.com,o=Your-Organization,c=US and entries for bar.com under ou=bar.com,o=Your-Organization,c=US. You would then add the following to your Cistron radiusd realms file:


foo.com  LOCAL  basedn=ou=foo.com,o=Your-Organization,c=US
bar.com  LOCAL  basedn=ou=bar.com,o=Your-Organization,c=US

This patch is known to apply cleanly to the latest Cistron radiusd release, cistron-radiusd-1.6.4.
